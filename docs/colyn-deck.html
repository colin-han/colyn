PART · 01

瓶颈

AI 能并行工作，但你不能——生产力被注意力带宽卡住了。

你有 3 个功能要做，3 个 Claude Code 在等你下指令——为什么还要一个一个来？
01 · 想并行，但又来了新问题

Clone 三份代码仓库？一系列新问题

💾

存储空间浪费

同一份代码 Clone 三份，磁盘空间三倍占用。项目大了之后尤为明显

🔌

服务端口冲突

三个 Dev Server 同时跑，端口打架。手动分配 3001、3002、3003，容易出错

🔀

合并代码困难

三份独立的 Git 仓库，合并代码需要通过远端仓库中转，费时费力，极易出错

01 · Worktree 能解决部分问题

Git Worktree 解决了存储，但还有门槛

$ git worktree add ../my-project-auth feature/auth
$ git worktree add ../my-project-tasks feature/tasks
$ git worktree add ../my-project-dashboard feature/dashboard
  • ▸命令冗长易错 — 路径、分支名手动指定，拼错就创建失败
  • ▸端口冲突要手动管 — 没有自动分配，改 .env 全靠自己
  • ▸依赖要重新装 — 每个 worktree 的 node_modules 独立，手动 npm install
  • ▸终端窗口记不住 — 开了三个终端，哪个是哪个？
  • ▸用完需要手动清理 — git worktree remove 容易忘，残留目录和分支堆积
PART · 02

初识 Colyn

五分钟安装，零配置上手——什么都没配，就能用了。
02 · 安装与初始化

安装到就绪，三步搞定

# 安装 Colyn
$ npm install -g colyn-cli
$ colyn setup
$ source ~/.zshrc
# 初始化当前项目
$ colyn init -p 3000
✓ 项目初始化完成
my-task-app/ ├── .colyn/ ├── my-task-app/ # 主分支 (PORT=3000) │ ├── src/ │ ├── package.json │ └── .env.local # PORT=3000 └── worktrees/ # Worktree 目录
  • ▸零配置 — 项目名从目录推断，端口写入 .env.local，自动检测 package.json
02 · 一条命令，就绪

colyn add，自动搞定一切

$ colyn add feature/auth
✅ Worktree 创建成功！
ID: 1 分支: feature/auth
路径: worktrees/task-1 端口: 3001
⠿ 安装依赖... ✓ 依赖安装完成
📂 已切换到: worktrees/task-1
  • ▸端口自动分配 — 3001、3002、3003…不冲突
  • ▸依赖自动安装 — 进去就能 npm run dev
  • ▸目录自动切换 — Shell 集成帮你 cd，无需手动操作

PART · 04

其他功能

修复、补全与多技术栈——Colyn 不只是 Node.js 的工具。
04 · 搬家也不怕

colyn repair，一键修复

  • ▸项目目录移动了 — Git worktree 的路径引用全部失效
  • ▸.env.local 损坏 — 端口配置丢失
  • ▸孤儿 worktree 目录 — 残留目录占用空间
$ colyn repair
✔ 检查主分支 .env.local...
✔ 修复 git worktree 连接...
✔ 检测并修复孤儿 worktree 目录...
✓ 修复完成！
一条命令，自动修复所有路径引用和环境变量文件。
04 · Tab 补全，效率翻倍

colyn setup 自动安装，Tab 即补全

04 · 按需定制

colyn set/get，灵活配置

$ colyn set lang zh-CN
✓ 配置已设置：lang = zh-CN (用户)
$ colyn set commands.merge.build false
✓ 配置已设置：commands.merge.build = false (项目)
$ colyn get lang
zh-CN
$ colyn unset verbose
✓ 已删除 verbose (用户)
  • ▸语言设置 — lang：中文 / 英文
  • ▸命令行为 — commands.merge.*、commands.release.* 等覆盖默认行为
  • ▸系统命令 — systemCommands.npm、systemCommands.claude 自定义路径
  • ▸Todo 后端 — todo.backend 支持本地文件或 GitHub Issues
  • ▸作用域 — 用户级（全局）或项目级，项目级优先
04 · 不只是 Node.js

Colyn 支持多种技术栈

Node.js Python Java Spring Boot Gradle Xcode Mono Repo
  • ▸自动检测 — 识别 package.json、pom.xml、requirements.txt 等项目文件
  • ▸端口自适应 — Node.js 默认 3000，Spring Boot 默认 8080，自动写入对应配置文件
  • ▸Mono Repo — 自动扫描子目录，每个子项目独立管理工具链
💡 无论你用什么语言开发，Colyn 的并行 Vibe Coding 工作流都是一样的——add → vibe → merge。

从"一个人写代码"到"指挥一支 AI 团队"，中间只差一个 colyn init

Colyn——让独立开发者也能拥有并行 Vibe Coding 的超能力。
