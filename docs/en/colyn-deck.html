PART · 01

Bottleneck

AI can work in parallel, but you can't — productivity is capped by your attention bandwidth.

You have 3 features to build and 3 Claude Codes waiting for instructions — why do them one at a time?
01 · Want to go parallel, but new problems arise

Clone three copies of the repo? A host of new problems

💾

Wasted Storage

Same code cloned three times, triple the disk usage. Especially noticeable as projects grow

🔌

Port Conflicts

Three dev servers running at once, ports collide. Manually assigning 3001, 3002, 3003 is error-prone

🔀

Merge Headaches

Three separate Git repos, merging code requires going through a remote — time-consuming, error-prone

01 · Worktree solves some problems

Git Worktree solves storage, but still has barriers

$ git worktree add ../my-project-auth feature/auth
$ git worktree add ../my-project-tasks feature/tasks
$ git worktree add ../my-project-dashboard feature/dashboard
  • ▸Verbose, error-prone commands — Paths and branch names manually specified, typos cause failures
  • ▸Manual port conflict management — No auto-assignment, editing .env is all on you
  • ▸Dependencies need reinstalling — Each worktree has independent node_modules, manual npm install
  • ▸Terminal windows are hard to track — Three terminals open, which is which?
  • ▸Manual cleanup needed — git worktree remove is easily forgotten, leftover dirs and branches pile up
PART · 02

Getting Started with Colyn

Five minutes to install, zero config to get started — no configuration needed, just works.
02 · Install & initialize

Install to ready, three steps

# Install Colyn
$ npm install -g colyn-cli
$ colyn setup
$ source ~/.zshrc
# Initialize current project
$ colyn init -p 3000
✓ Project initialized
my-task-app/ ├── .colyn/ ├── my-task-app/ # Main branch (PORT=3000) │ ├── src/ │ ├── package.json │ └── .env.local # PORT=3000 └── worktrees/ # Worktree directory
  • ▸Zero config — Project name inferred from directory, port written to .env.local, auto-detects package.json
02 · One command, ready to go

colyn add, handles everything automatically

$ colyn add feature/auth
✅ Worktree created successfully!

PART · 04

More Features

Repair, completion & multi-stack support — Colyn is more than just a Node.js tool.
04 · Not afraid of moving

colyn repair, one-click fix

  • ▸Project directory moved — All Git worktree path references are broken
  • ▸.env.local corrupted — Port configuration lost
  • ▸Orphaned worktree directories — Leftover dirs wasting space
$ colyn repair
✔ Checking main branch .env.local...
✔ Fixing git worktree connections...
✔ Detecting and fixing orphaned worktree directories...
✓ Repair complete!
One command, automatically fixes all path references and environment variable files.
04 · Tab completion, doubled efficiency

colyn setup auto-installs, Tab to complete

04 · Customize on demand

colyn set/get, flexible configuration

$ colyn set lang en
✓ Config set: lang = en (user)
$ colyn set commands.merge.build false
✓ Config set: commands.merge.build = false (project)
$ colyn get lang
en
$ colyn unset verbose
✓ Deleted verbose (user)
  • ▸Language setting — lang: Chinese / English
  • ▸Command behavior — commands.merge.*, commands.release.* override defaults
  • ▸System commands — systemCommands.npm, systemCommands.claude custom paths
  • ▸Todo backend — todo.backend supports local files or GitHub Issues
  • ▸Scope — User-level (global) or project-level; project takes priority
04 · More than just Node.js

Colyn supports multiple tech stacks

Node.js Python Java Spring Boot Gradle Xcode Mono Repo
  • ▸Auto-detection — Recognizes package.json, pom.xml, requirements.txt and other project files
  • ▸Adaptive ports — Node.js defaults to 3000, Spring Boot to 8080, auto-writes to the right config file
  • ▸Mono Repo — Auto-scans subdirectories, each sub-project managed independently
No matter what language you use, the Colyn parallel Vibe Coding workflow stays the same — add -> vibe -> merge.
